データベースを扱ってみたいなで作ったのがBlog機能です
データベースにはSupabaseを使い、他はすべてここで処理できるようにと考えていました。
とりあえずデータの読み出しさえできればここで表示する分には申し分ないので
そこだけ作ろうと思って公式ドキュメントからパクって持ってきて作ったのが以下のコード
    <html>
        ...
        <script src="https://unpkg.com/@supabase/supabase-js@2"></script>
        <script>
            import {createClient} from '@supabase/supabase-js@2'
            Client = supabase.createClient("https://****.supabase.co", "sb_publishable_*****");
        </script>
        ...
    </html>
しかし、import ...の行で外部からインポートできません的なエラーが吐かれます
Google君に聞いてみたところmoduleを付けると大丈夫と言われたのでそう書き変えました。
しかし、CDNのものにmoduleタグをつけてもダメです
んで、Pythonを使ったAPI Serverを建ててそこを介する処理にしようとしましたが、面倒くさくってやめました。(この間2週間)
そして、ネットの海をさまよっていると、こんなものを発見

<!DOCTYPE html>

    <script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@latest"></script>

<script>
class SupabaseClientComponent extends HTMLElement {
    constructor() {
        super();
        this.supabaseUrl = '';
        this.supabaseKey = ''; 
        this.supabase = null;
    }

    connectedCallback() {
        this.initializeSupabase();
    }

    initializeSupabase() {
        this.supabase = supabase.createClient(this.supabaseUrl, this.supabaseKey);
        console.log('Supabase client has been initialized:', this.supabase);
        // Now you can use this.supabase to make requests to your Supabase project.
    }
}

// Define the new element
customElements.define('supabase-client', SupabaseClientComponent);

</script>
</html>

https://github.com/supabase/supabase/issues/21261#issuecomment-1946066213より
importはいらない子だったということです。ありがとう海外ニキ。
CDNインストール(scriptタグを使う、Web向け)を行うのならばimportを使わないように！
公式ドキュメントから何も考えずにコピペしてくると必ずブチ当たる間違いだと思います、気を付けてください！
ここさえ乗り越えたらあとは公式ドキュメントを使って何でもできます、つまり無敵です
